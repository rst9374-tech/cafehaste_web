# 개발 에이전트 지침서 (Agent Instructions)

- **기본값 무조건 승인**: 작업을 진행할 때 지속적인 동의 구득이나 질의 단계를 거치지 않고, 시스템 변경 및 코드 수정을 기본적으로 **무조건 수락 및 자동 승인(Auto-approve)하여 신속하게 수행**합니다.
- **원격 빌드 및 배포 금지**: 원격 클라우드 빌드 및 실서버 배포(`npm run deploy` 등)는 절대 가동하지 않으며, 오직 로컬에서의 코드 수정과 로컬 빌드 검증(`npm run build`)만 허용합니다.
- **사전 검토의견 제시 (질의/요청)**: 사용자가 질문을 하거나 개발 변경사항을 요청하면, 실제 빌드(컴파일 및 적용)를 수행하기 전에 에이전트의 검토 의견(기술 피드백, 보정 의견 등)을 반드시 먼저 제시한 후 구현에 들어갑니다.
- **최대한 간결하게 답변**: 설명은 한두 줄 이내로 최소화하고 즉시 반영합니다.
- **안전한 기본 적용**: 가장 추천하는 최적의 방식으로 바로 적용하되, 치명적인 문제(Side-effect)가 있는 경우에만 아주 간결하게 사전에 보고합니다.

[최종 마스터 본] 헤이스트(HASTE) AI 에이전트 개발 지침서
Vite + Node.js + Supabase Infrastructure 환경에 특화된 경량화 및 비용 최적화 설계 규약입니다.

---

📌 1. 리눅스 호환을 위한 소문자(lowercase) 기반 파일 및 컴포넌트 명명 구조 규약
- 상세 기술 사양 및 명명 양식은 에이전트 전용 스킬인 `haste-code-standards`를 따릅니다.

📌 2. Supabase Storage 단일 버킷 격리, 이미지 Canvas 압축 및 DB-스토리지 트랜잭션 롤백 무결성 규약
- 상세 이미지 압축 알고리즘, 트랜잭션 및 폴백 사양은 `haste-code-standards` 스킬에 의거하여 제어합니다.

📌 3. 초고속 DB 커넥션 해제, 실시간 폴링 지양 및 비용 0원 기반 인메모리 싱글톤 캐시 제어
- 상세 커넥션 풀 반환 프로토콜 및 벌크 API 연동 규칙은 `haste-code-standards` 스킬을 준수합니다.

📌 4. 코드 경량화 최적화 및 500줄 이하 파일 청결도 유지 규약
- **500줄 초과 방지 및 분할**: 모든 소스코드 파일은 단일 파일 500줄 이하를 엄격히 준수합니다. 초과 시 비동기 제어 로직 등은 커스텀 훅이나 독립 모듈로 분할 추출합니다.
- **공통 헬퍼/훅 추출**: 여러 파일에서 중복되는 API Fetch 요청, Toast 호출, 반복 로직 등은 `adminFetch`와 같은 공통 유틸/훅으로 통합하여 코드 중복을 최소화합니다.
- **CSS 테마 규격화**: 복잡한 Tailwind CSS 중복 선언이나 인라인 스타일(style 속성)을 지양하고, `index.css` 등에 `.admin-table`, `.admin-btn-action`, `.admin-modal-overlay` 등 표준 클래스를 정의하여 공통 규격으로 대체합니다.
- **데드 코드 상시 소거**: 외부에서 쓰이지 않는 exports 지시어와 미사용 선언/백업 파일은 발견 즉시 안전하게 정리 및 삭제 처리합니다.
- 상세 규격은 `haste-code-standards` 스킬의 코딩 표준 사양을 따릅니다.

📌 5. AI API 비용 방어 및 데이터 필드 카멜케이스(camelCase) 통합 규약
- AI 토큰 디펜스 기법 및 camelCase-snake_case 맵핑은 `haste-code-standards` 스킬의 구현체를 따릅니다.

📌 6. 자동화된 빌드 이력 관리 (RELEASES.md)
- **빌드 이력 추적**: 간단한 수정사항은 기록하지 않으며, DB 스키마 수정 등 시스템 큰 변화 발생 시 기록합니다.

📌 9. 사용자 및 어드민 페이지 내 한글 표준 용어 준수 규칙
- `선배 점주` 표기 지양 ➔ **베테랑 점주**로 통일하여 노하우 공유 및 안내 뉘앙스를 전문화합니다.
- **보안을 위한 `WMF` 키워드 사용 금지**: UI 및 소스코드 내 모든 `WMF` 표기는 보안 규정 준수를 위해 예외 없이 **커피머신**으로 일괄 치환 처리합니다.
- **자연스러운 구어체 지향**: 마케팅 문구 및 서비스 안내 텍스트 작성 시, 기계적인 IT 번역투나 과장되고 생소한 단어(예: 구원하는, 영위하는, 고체 풍미 등)의 남용을 전면 차단하며, 실제 사람이 대화하듯 매끄럽고 신뢰성 있는 자연스러운 한국어 구어를 상시 준수합니다. 특히 **"100%"**와 같이 극단적이고 절대적인 수치적 과장 표현 역시 브랜드의 정중함과 품격을 저해하므로 사용을 엄격히 배제하며, 정중하고 유연하게 완화하여 표현합니다. 또한 **"`유통 마진이 없다`"** (일절 없다, 유통마진 무관 등)는 식의 마진/수수료에 대한 직설적인 문구는 브랜드의 신뢰성을 저해하므로 사용을 금지하며, 자율 소싱 보장 또는 매장 비용 절감이라는 취지로 우회하여 간접 서술합니다. 가독성을 해치는 불필요한 괄호 표기(예: 무상(0원) 등) 역시 제거하여 담백하게 문장을 마감합니다.

📌 10. 관리자 명령 및 제어 접두사 규칙
- 에이전트는 장착된 `haste-agent-command-parser` 스킬의 커맨드 분기 사양에 의거하여 배포 및 코드 최적화를 자동으로 제어합니다.

📌 11. HASTE PLAYLIST (음악 서비스) 및 회원 접근 권한 격리 규약
- **이원화 회원 격리**: 음악 일반 회원(`music_users`, 리스너)은 카페헤이스트의 멤버십 전용 라운지, Q&A 게시판, 커피머신 제어 등 카페 관련 어떠한 서비스 및 API에도 접근할 수 없도록 철저히 차단(403 Forbidden 및 접근 제한)해야 합니다.
- **점주 권한 우대**: 유료 멤버십 가맹점주(`web_membership_users`)는 음악 서비스(BGM 플레이어 재생 및 음악 커뮤니티 등록/피드백 등)를 추가 비용 없이 모두 이용할 수 있는 통합 최상위 권한을 지닙니다.
- **DB 및 물리 파일명**: 음악 서비스와 카페 핵심 서비스 간의 상호 오염을 차단하기 위해 데이터베이스 테이블명은 각각 `music_` 및 `web_` 접두사로 엄격히 격리하며, 프론트엔드 컴포넌트는 `music_` 및 `home_`/`membership_` 등으로 네이밍 규칙을 분리합니다.
- **유튜브 BGM 제목 자동 정제**: 유튜브 링크를 통한 BGM 등록 시, 노이즈 키워드(`[Playlist]`, `(광고없음)`, `연속재생` 등)를 자동 제거하고, 브랜드 접두사 `Haste_` 또는 `Haste BGM: `을 강제 결합하여 브랜딩 제목으로 자동 가공합니다.
- **공통 홍보문구 결합**: 등록되는 BGM의 설명에는 헤이스트 공통 홍보문구("헤이스트 가맹점 전용 아날로그 BGM 서비스로, 방문하는 손님들과 점주님 모두에게 깊이 있고 편안한 무드의 하루를 선사합니다.")가 기본 결합되도록 유도합니다.
- **동적 SEO 메타 태그**: 플레이리스트 및 BGM 페이지를 SNS(카카오톡, 페이스북 등)에 공유할 시, 해당 곡의 정제된 타이틀과 유튜브 썸네일 이미지가 오픈그래프 메타 태그(`og:title`, `og:image`, `og:description`)에 실시간 반영되어 노출되도록 강제 연동합니다.

📌 12. 헤이스트(Haste) 브랜드 AI 음원 제작 지침서
- 에이전트는 장착된 `haste-music-generator` 스킬의 브랜드 음원 특수 규칙, 프롬프트 템플릿 매핑 및 7대 응답 포맷 요구사항에 의거하여 AI 음원을 제작합니다.
