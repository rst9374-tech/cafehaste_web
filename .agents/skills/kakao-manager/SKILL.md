# 📋 HASTE Kakao Post Processor Skill

이 스킬은 사용자가 무인 카페 실전 운영 노하우 등의 원본 꿀팁 글을 제공했을 때, 이를 규격화된 지침서 포맷으로 자동 변환하고 작성자 지점 매핑 및 글쓴날짜 동기화를 수행하여 Supabase DB에 실시간으로 반영하는 작업을 자동화합니다.

## 🛠️ 주요 기능 및 규칙

1. **지점별 작성자(member_id) 매핑**:
   - 본문 내 `작성자: XXX` 지점을 추출하여 `web_membership_users` 테이블에 해당하는 가상 계정이 있는지 조회하고, 없으면 표준 규격(인증 완료, 비밀번호 해시, 가상 이메일/휴대폰 등)으로 가상 점주 계정을 생성한 후 `member_id`로 바인딩합니다.
   
2. **날짜 동기화 및 제목 정제**:
   - 제목 또는 본문에서 날짜 정보(예: `2025.02.16` 또는 `2024.12.15 ~ 2025.04.12`)를 파싱합니다.
   - 제목에서 날짜 부분을 깔끔하게 제거하여 정제합니다.
   - 실제 날짜를 DB의 `created_at` 및 `updated_at` (글쓴날짜) 값으로 강제 설정합니다. (범위일 경우 마지막 날짜 기준)

3. **대괄호 구조화 지침서 템플릿 자동 적용**:
   - 뷰어에서 굵은 글씨 및 블록 처리가 되도록 본문 내용을 다음 4가지 대괄호 블록 구조로 가공합니다:
     - `[1. 발생 원인 및 문제점]`
     - `[2. 필요 준비물 및 도구]` (또는 `[2. 해결 노하우]`)
     - `[3. 상세 제작 및 설치 순서]` (또는 `[3. 상세 청소 및 정비 순서]`)
     - `[4. 설치 효과 및 주의사항]` (또는 `[4. 효과 및 관리 주의사항]`)

4. **보안 키워드 치환**:
   - 본문 내 모든 WMF 단어는 예외 없이 `커피머신`으로 안전 우회 치환하여 DB에 등록합니다.

  2. `c:\Users\김성규\Desktop\HASTE-Company\src\cafehaste-web\.agents\skills\kakao-post-processor\scripts\analyze_chats.ts`

       - (참고: `--all` 플래그를 붙여 실행하면 대기 중인 모든 꿀팁 후보를 요약 표 형태로 일목요연하게 출력합니다.)

## 💡 호출 및 작업 규칙
사용자가 **"카톡방 분석해줘"** 혹은 **"여기 디렉토리에서 카톡 분석해줘"**라고 요청하며 특정 경로를 제공하는 경우:
1. `analyze_chats.ts` 스크립트를 실행하여 디렉토리 내 `KakaoTalkChats.txt`에서 아직 처리되지 않은 이미지와 연관된 팁 후보들을 스캔합니다.
2. 스캔 결과를 토대로 가장 가치 있는 첫 번째 꿀팁 후보를 보고서 포맷(제목, 작성자, 날짜범위, 상세내용 - 원인/조치결과)으로 사용자에게 제안합니다.
3. 사용자가 **"승인"**하면, 임시 포스트 파일(`temp_tip_post.txt`)을 생성한 후 `process_post.ts` 스크립트를 실행하여 DB 등록 및 이미지 첨부를 완료합니다.
4. 등록이 끝나면 `mark_processed_tip` 방식으로 원문 `KakaoTalkChats.txt` 내 해당 대화 앞에 `// [등록완료_YYYYMMDD]` 주석 처리를 하고, 로컬 폴더의 물리 이미지 파일명 뒤에도 `_uploaded.jpg`를 붙여 rename함으로써 나중에 중복 처리되는 것을 방지합니다.

## 📌 대화록 가공 및 주입 행동 규약

에이전트는 카톡 대화 추출 및 DB 주입 스킬을 실행할 때 아래 사항들을 엄격히 적용해야 합니다:

1. **지점별 작성자(member_id) 및 90000대 ID 매핑:**
   - 대화록 수집 게시글의 작성자는 반드시 `web_kakao_members`에 등록된 90000대 가상 ID로 매핑하여 DB에 INSERT해야 합니다. 그래야만 프론트엔드 UI 목록에서 카톡 아이콘 표시 배지가 정상 유지됩니다.
   - 일치하는 지점명이 없을 경우, `web_kakao_members` 테이블 규칙(ID: MAX+1, owner_name: '베테랑 점주', approval_status: '인증 완료')에 맞춰 신규 가상 회원 행을 동적으로 생성한 후 매핑합니다.

2. **광범위 융합 및 중복 배제:**
   - 단편적인 조각 대화 단위로 글을 쪼개어 여러 개 발행하지 말고, 최소 일주일 이상의 범위를 훑어 동일 주제에 관한 모든 질답 및 팁을 하나의 통합 노하우 게시글로 병합합니다.
   - 대화 범위 내 신규 회원의 단순 입장 인사나 점주 인증 관련 사담, 영수증 캡쳐 등은 팁 추출 후보 및 본문 텍스트 구성에서 100% 원천 배제합니다.

3. **대화 범위 이미지 자동 매핑:**
   - 대화 범위 내에서 점주들이 업로드한 이미지(`<사진 읽지 않음>` 코드)가 존재할 경우, 대화 전송 시각 기준 ±15분 오차 범위 내의 이미지 파일의 수정 시각을 색출하여 `/uploads/board/` 경로로 리사이징 압축 복사하고 본문 하단 첨부파일 영역에 매핑합니다. (CCTV 캡쳐 인증샷 등 무관한 사물 노이즈 이미지는 제외)

# 카카오톡 대화방 수동 주입 기능 스킬 (kakao-injection)

본 지침은 사용자가 카카오톡 대화록을 DB에 수동 주입(팁게시판 등)을 요청할 때 가동하는 전문 태스크 스킬입니다.

## 🛠️ 실행 및 가공 절차

### 1. 트리거 감지 및 탐색
* 사용자가 *"카톡주입 YYYY-MM-DD 날짜부터 시작해"* 라고 입력하면 가동합니다.
* 에이전트는 `cafehaste-etc/kakao` 디렉토리 하위의 `.txt` 파일 및 이미지 파일들을 탐색합니다.

### 2. 시간 필터링
* 대화록 파일에서 지정된 YYYY-MM-DD 시점 이후의 대화 블록만 요약 대상으로 자동 추출/분리합니다.

### 3. 자동 요약 및 가공 (보안 치환)
요약 작성 시 `SKILL.md (본 지침서)` 내에 정의된 다음 규칙들을 준수합니다:
* **가상 회원 매핑**: 90000대 ID를 부여하며, 작성자는 '베테랑 점주'로 처리합니다.
* **문답 포맷 및 정제**: 가상 회원 간의 질문과 답변 형식으로 정비합니다.
* **금지어 보안 치환**: 
  * '본사' ➔ '가맹지원팀' 등으로 내부 보안 키워드 치환 적용.
* **게시글 연동**: 유사 게시글 링크 연동 가이드를 반영합니다.

### 4. CLI 스크립트 구동
* 정제가 완료되면 임시 `payload.json`을 구성합니다.
* `npm run tips:cli` 명령어를 사용하여 로컬/실서버 DB의 `test` 카테고리에 글을 최종 게시합니다.

---

## 1. 날짜 기반 대화 파싱 및 필터링 규칙
* **명시적 날짜 필터**: 사용자가 *"카톡주입 YYYY-MM-DD 날짜부터 시작해"* 라고 명시한 시각 이후의 대화 블록만 텍스트 파일에서 읽어와 요약 대상으로 삼습니다.
* **타임스탬프 추출**: 정제된 게시글의 생성 날짜(`created_at`)는 대화방 텍스트에 나타난 **실제 대화 시간**을 파싱하여 타임슬립 형태로 등록해야 합니다.

## 2. 가상 회원 맵핑 및 가입 규칙
* **ID 대역 격리**: 카톡 제보 점주들은 일반 회원들과 구별되도록 **`90000`번대 고유 ID**를 부여합니다.
* **작성자 고정**: 제보 점주들의 성함은 모두 **`베테랑 점주`**로 일관되게 치환하여 가입 및 게시판에 표기합니다.
* **회원 무결성**: 글 작성 시 반드시 `web_kakao_members` 테이블 및 `local_kakao_members.json` 백업 리스트에 등록된 회원 ID(`member_id`)를 활용하여 참조 무결성을 보장합니다.

## 3. 정보 요약 및 고도화 가공 규칙
* **소제목 포맷팅**: 본문의 단락 소제목은 프론트엔드 모듈이 골드 컬러 굵은 서체로 변환할 수 있도록 반드시 **`[소제목 내용]`** 형태로 마크다운을 빌드합니다.
* **문답 출처 명시**: 정보 제공에 기여한 질문 지점과 조치 지점명을 대화 내용에서 파악하여 본문 소제목 바로 아래에 삽입합니다.
  - *예시: `(질문 지점: 군포금정점 / 조치 해결 지점: 김포구래럭스나인점)`*

## 4. 금지어 보안 필터링 규칙
작성하는 글의 본문 및 제목 내에 가맹본부 오인적 단어나 특정 브랜드명이 무단 노출되지 않도록 반드시 아래 변환 테이블에 맞춰 사전 치환합니다.

| 검출된 금지 단어 | 대체 반영할 표준 단어 | 변환 적용 이유 |
| :--- | :--- | :--- |
| **본사** | **가맹지원팀 / 플랫폼 운영팀** | 가맹점주 주도형 오픈 커뮤니티 지향성 수호 |
| **기사 / 엔지니어 / 임원** | **기기 제조사 서비스 / 디지털 매뉴얼** | 본사 소속 직원의 무분별한 지칭 차단 및 책임 한계 구별 |
| **WMF** | **커피머신** | 특정 제조사 하드웨어 기종 노출 방지 (보안) |
| **무상 / 무료** | **기본 지원 / 제공** | 점주 오인으로 인한 법적 분쟁 리스크 사전 완화 |
| **100% / 완벽** | **기본 / 상당수 / 효과적** | 수치적 과장 표현 순화로 신뢰도 확보 |

## 5. 유사 게시글 교차 연동 규칙
* 신규 꿀팁을 주입할 때 카테고리 내에서 기존 게시글의 제목 키워드를 매칭하여, 본문 맨 아래에 아래 형식으로 추천 링크를 자동 생성해 연동해 주어야 합니다.
  - *예시: `[비슷한 글: [커피머신] 48시간 우유 세척 해결 가이드](/board/167)`*

## 6. 대화록 사진(<사진 읽지 않음>)의 물리적 이미지 매핑 규칙
* **현상**: 카톡 대화록을 텍스트로 내보낼 때, 미디어 파일 만료 등으로 인해 이미지 메시지가 대화록 상에는 `<사진 읽지 않음>` 또는 `[사진]` 으로 텍스트가 뭉개집니다. 또한 로컬 미디어 폴더(`img`) 안의 사진명들은 모두 해시값(`0a10d28b...jpg`)으로 치환되어 이름만으로는 구별이 어렵습니다.
* **매핑 솔루션 (시간순 정렬 매핑 규칙)**:
  - 카톡방 미디어가 백업 및 업로드되는 과정에서 **로컬 이미지 파일의 수정 시간(mtime)의 오름차순(과거 ➔ 최신) 정렬 순서**는 대화록 상에서 각 사진 메시지가 올라온 **원래 전송 시간 순서와 100% 동일(비례)**하게 매치됩니다.
  - 따라서, 대화록 내에서 목표 사진 메시지가 그날 몇 번째 순서로 등장했는지를 파악한 후, 로컬 `img` 폴더를 `mtime` 오름차순 정렬한 파일 리스트의 동일한 상대적 순서(인덱스)의 해시 파일을 매칭하면 100% 신뢰도로 정확히 식별해낼 수 있습니다.
* **수행 원칙**: 
  - OCR(광학 문자 인식) 등 대량의 CPU 연산을 요구하여 사용자 PC 성능을 버벅거리게 만드는 무거운 탐색 방식은 일절 배격합니다.
  - 오직 대화 기록의 타임라인 등장 순서와 파일 시스템의 수정 시각 정렬 순서를 이용한 이 수학적·논리적 순서 비례 규칙을 최우선으로 적용하여 0.1초 만에 스마트하게 매핑을 도출해내야 합니다.

## 7. 주입기 자동 감지 핵심 키워드 리스트 (총 27개)
단톡방 대화록에서 주입기가 자동으로 탐지하고 분류를 파싱하는 기준이 되는 최종 키워드 맵입니다.

| 분류 | 감지 키워드 | 목적 및 동작 |
| :--- | :--- | :--- |
| **제목 대괄호 분류** | `커피머신`, `기기운영`, `매장운영` | 글 제목 맨 앞에 알맞은 접두사 대괄호 접두사(예: `[커피머신]`) 자동 삽입 |
