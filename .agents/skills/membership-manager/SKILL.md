# 회원/권한 관리 스킬 (haste-membership-manager)

본 지침은 HASTE 플랫폼의 가입 회원 유형(일반 리스너 vs 가맹점주 vs 관리자)에 따른 구조 정의 및 권한 통제 로직을 규율하는 태스크 스킬입니다.

## 📌 1. 이원화 회원/권한 격리 아키텍처
* **일반 음악 리스너 (`music_users` 테이블)**:
  - 권한: 음악 재생 및 커뮤니티 피드백 참여만 허용.
  - 제한: 점주용 웹 대시보드(`/control`), 어드민 패널 및 원격 커피머신 제어 API 접근 차단.
* **가맹점 경영주 (`web_membership_users` 테이블)**:
  - 권한: 대시보드 진입, 키오스크 메뉴 및 옵션 매핑 관리, 레시피 코드 전송, 실시간 로컬 로그 모니터링 허용.
* **가상 점주 ID 대역 격리**:
  - 카톡 제보 점주들은 일반 회원들과 구별되도록 **`90000`번대 고유 ID**를 부여합니다.
  - 작성자 성함은 상시 **`베테랑 점주`**로 표준화 치환하여 처리합니다.

## 📌 2. 점주 가입 상태 천이 규칙 (State Transition)
점주 회원의 심사 및 라이선스 작동 상태는 다음 3단계 표준 용어 및 상태를 엄격히 준수합니다:
1. **인증 대기**: 
   - 회원 가입 신청 후 관리자의 심사를 대기하는 최초 상태.
2. **인증 완료**: 
   - 가입 심사가 정식 통과되어 정상 가동 중인 상태. (상태 변경 즉시 고유한 라이선스 키가 자동 생성되어 점주 디지털 카드에 매핑 노출됩니다)
3. **가동 정지 / 가동 정지중**: 
   - 구독 기간 만료, 미정산, 또는 불법 라이선스 유출 탐지 시 강제 제한된 상태. (로컬 API 인증이 즉시 차단됩니다)

## 📌 3. 권한 정합성 및 캐시 무효화
* 점주 정보 및 라이선스 권한 상태가 변경되는 즉시, 인메모리 싱글톤 캐시(`storeVerifyCache[storeId]`)를 삭제(`delete`)하여 점포의 키오스크 및 로컬 서버가 수정된 권한 정보를 실시간으로 갱신 조회하도록 보장해야 합니다.
