# 메뉴/레시피 총괄 관리 스킬 (haste-menu-manager)

본 지침은 메뉴 등록, 이름 표준화, 이미지 매핑 및 스토리지 적재 작업을 일원화하여 자동화하는 태스크 스킬입니다.

## 📌 1. 음료 표준 네이밍 변환 규칙 (web_drink_name_standards)
상품이름 및 메뉴판 노출 한글명 작성 시 다음 표준 원칙을 적용합니다:
* **디카페인 음료**: 일반명 앞에 **`(D)`** 또는 **`디카페인 `**을 강제 결합합니다. (예: `HOT_(D)아메리카노`, `아이스 디카페인 카페라떼`)
* **프리미엄 음료**: 일반명 앞에 **`(P) `** 또는 **`프리미엄 `**을 결합합니다.
* **아메리카노 명칭**: 아메리까노, 아메리카 등의 외래어 오기입은 에이전트가 탐지하여 **`아메리카노`**로 자동 정정합니다.

## 📌 2. 메뉴 이미지 매칭 및 스토리지 업로드 (menu-image-uploader)
수동 이미지 업로드 시, 클라이언트 단에서 한글 음료명 및 온도/원두 정보를 분석하여 표준 파일명으로 자동 리네이밍 매칭합니다:
* **디카페인 원두 (한글명에 '디카페인' 혹은 '(D)' 포함)**: 디카페인 띠지 이미지 매치 (`HOT_(D)제품명.png` / `ICED_(D)제품명.png`)
* **프리미엄 원두 (한글명에 '프리미엄' 혹은 '(P)' 포함)**: 프리미엄 띠지 이미지 매치 (`HOT_(P)제품명.png` / `ICED_(P)제품명.png`)
* **일반 표준 원두**: 일반 표준 이미지 매치 (`HOT_제품명.png` / `ICED_제품명.png`)
* **수량 구성 비율**: 아메리카노(각 17개씩) 및 라떼(각 30개씩) 부문의 일반, 디카페인, 프리미엄 분류별 수량은 100% 일치 대칭을 유지합니다.

## 📌 3. PostgreSQL Public RLS 임시 제어 업로드 가이드
로컬 환경에서 익명(Anon) Key의 쓰기 RLS 제한이 걸리는 경우, PostgreSQL 커넥션을 통해 임시로 RLS 정책을 생성 및 회수합니다:
```sql
-- 1. 임시 public 업로드 RLS 정책 추가
CREATE POLICY temp_allow_all ON storage.objects 
  FOR INSERT WITH CHECK (bucket_id = 'cafehaste-bucket');

-- 2. 업로드 작업 수행 (스크립트 실행)

-- 3. 업로드 완료 즉시 정책 삭제 및 회수
DROP POLICY temp_allow_all ON storage.objects;
```
