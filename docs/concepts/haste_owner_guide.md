### 📋 헤이스트(HASTE) 전산 로그(Log) 시스템 가이드북

| 대구분 | 중구분 | 핵심 사양 | 로그 처리 규칙 및 파싱 분석 포맷 |
| :--- | :--- | :--- | :--- |
| **1. 수집 파이프라인** | 인메모리 버퍼링 | `verifyApiLogs` 세션 | • 커피머신 로컬서버 가동 시 발생하는 트래픽 검증 로그를 속도 유지를 위해 1차적으로 서버 RAM 내부의 임시 배열 버퍼에 적재 |
| | 디스크 동기화 (Flush) | `/admin/logs/flush` | • 인메모리 적재 로그를 영구 디스크 텍스트 파일로 기록하는 동기화 트리거 실행 (트래픽 로그 벌크 디스크 기록) |
| | 실시간 메모리 소거 | `/verify-logs/clear` | • 세션 안정성을 위해 필요 시 실시간 인메모리 로그 배열을 즉시 완전 초기화(Clear) |
| **2. 저장 및 보존 정책** | 로그 저장 경로 | `logs/` 디렉토리 | • 프로젝트 루트 하위의 `logs/` 폴더 내에 일 단위 텍스트 파일로 격리 저장 |
| | 로그 파일명 규칙 | `haste_api_log_YYYY-MM-DD.txt` | • 한국 표준시(KST) 날짜 기준으로 하루에 한 파일씩 자동 분할 보존 (`haste_api_log_2026-06-04.txt` 등) |
| | 보존 한도 | 최대 3개월 (90일) | • 디스크 과부하 및 용량 고갈 예방을 위해 일간 단위 로그는 3개월간 보존하며, API를 통해 날짜별 조회 및 다운로드 지원 |
| | 좌/우 로그 필터 분리 | leftLogs & rightLogs | • 좌측 로그보드에는 숫자 형식의 실제 매장 로그만 노출<br>• 우측 터미널 제어기에는 STORE- 형식의 가상 테스트 로그만 노출해 운영계/테스트계 관심사 격리 분리 |
| **3. 로그 텍스트 포맷** | 라인당 저장 규격 | 텍스트 인코딩 | • `[연동시각] [매장코드] [IP 정보] [승인판정] - 상세 로그 메시지`<br>➔ `[2026-06-04 10:25:46] [STORE-1001] [IP: 127.0.0.1] [APPROVED: PASS] - 정식 승인 통과 (강남본점 / PREMIUM)` |
| | 정규식 파싱 규칙 | RegExp 파서 | • 로그 로딩 및 차트/터미널 시각화를 위한 자동 정규식 파싱 매핑 적용:<br>➔ `/^\\[([^\\]]+)\\]\\s+\\[([^\\]]+)\\]\\s+\\[IP:\\s+([^\\]]+)\\]\\s+\\[APPROVED:\\s+([^\\]]+)\\]\\s+-\\s+(.*)$/` |
| **4. 조회 및 다운로드** | 실시간 로그 terminal | `/verify-logs` | • 어드민 패널 내 터미널 뷰어로 실시간/일별 로그를 역순(최신 로그가 상단)으로 스크롤 로딩 조회 |
| | 일별 다운로드 | `/verify-logs/download` | • 특정 일자(`date=2026-06-04`) 쿼리 매핑 시 해당 일자 로그 텍스트 파일 스트림 즉시 다운로드 제공 (Path Traversal 차단 방어선 적용) |

### 📋 헤이스트(HASTE) 로그 등급별 분석 및 대응 가이드

| 로그 등급 | 판정 상태 | 주요 발생 원인 및 로그 문구 예시 | 본사 시스템 영향 및 대응 조치 |
| :--- | :--- | :--- | :--- |
| 🟢 **정상**<br>`(PASS / INFO)` | **인증 승인 완료**<br>(정식 구동) | • `APPROVED: PASS`<br>• `정식 승인 통과 (지점명 / GRADE)`<br>• `[캐시 히트] 5분 초고속 패스구동 인증 통과`<br>• `인메모리 로그가 디스크에 성공적으로 반영되었습니다.` | • 커피머신 로컬서버 정상 가동 상태<br>• 7일 오프라인 구동 보안 토큰 발급 완료<br>➔ **조치 불필요 (안정 가동)** |
| 🟡 **주의**<br>`(WARN / EXPIRED)` | **기간 만료**<br>(임시 구동 유예) | • `APPROVED: FAIL` (`EXPIRED`) - allowOfflineGrace: true<br>• `EXPIRED: 유예 토큰 발급` (라이선스 만료)<br>• `타임스탬프 만료 (오차: Xms)` (동기화 오차)<br>• `[DB Healing] Restored broken bg_image...` (자가치유) | • 라이선스 유효기간 만료 상태<br>• 7일 임시 구동 유예 토큰 발급됨<br>➔ **경영주에게 라이선스 만료 경고 및 갱신/연장 안내 발송 요망** |
| 🔴 **위험**<br>`(FAIL / DANGER)` | **인증 차단**<br>및 **보안 위협** | • `APPROVED: FAIL` (`SUSPENDED`) - allowOfflineGrace: false<br>• `SUSPENDED: 본사 조치 차단 완료` (미정산 등 본사 락)<br>• `INVALID_KEY: 인증되지 않은 접근 시도` (인증키 오류)<br>• `[DDoS Shield Active] IP blocked...` (DDoS 디도스 경보)<br>• `Supabase Query Interceptor Failed...` (DB 커넥션 고갈) | • **본사 강제 정지** 또는 **외부 해킹/DDoS 위협** 발생<br>• 유예 토큰 발급 차단 (전산 완전 잠금)<br>➔ **DDoS 차단 IP 대역 모니터링, 해킹 시도 여부 점검 및 긴급 DB 커넥션 풀 복구 확인** |
