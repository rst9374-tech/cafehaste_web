# 실시간 검증기 및 로그 규격서 (Live Verifier & Log Specifications)

이 문서는 커피머신 로컬서버 가동 시 발생하는 트래픽 검증 로그의 수집, 일간 파일 저장 및 분석용 정규식 파싱 규격을 정의하는 전산 명세서입니다.

---

## 📌 1. 실시간 검증기(Live Verifier) 작동 규격
*   **1차 인메모리 적재**:
    - 커피머신 로컬서버가 전송하는 실시간 통신 데이터는 성능 방어와 응답 속도 보장을 위해 1차적으로 서버 RAM 내부의 임시 배열 버퍼(`verifyApiLogs`)에 저장됩니다.
*   **2차 디스크 동기화 (Flush)**:
    - 로컬 디스크의 영구 저장을 위해 `/admin/logs/flush` API를 기동하여 인메모리에 적재된 로그 데이터를 텍스트 파일로 동기화 기록(Flush)합니다.

---

## 📌 2. 로그 저장 및 보존 정책
*   **저장 경로**: 프로젝트 루트 하위의 `logs/` 디렉토리에 일 단위 분할 텍스트 파일로 저장합니다.
*   **파일명 규칙**: `haste_api_log_YYYY-MM-DD.txt` (KST 한국 표준시 기준 일자 분할 보존)
*   **보존 한도**: 디스크 고갈 방지를 위해 최대 3개월(90일)간 보존하며, 3개월이 경과한 파일은 자동 백업 후 안전 소거 처리됩니다.

---

## 📌 3. 로그 파일 텍스트 라인 포맷 (Text Line Format)
로그 파일의 한 줄당 기록 규격은 아래와 같습니다.

```text
[연동시각] [매장코드] [IP 정보] [승인판정] - 상세 로그 메시지
```

### RegExp 정규식 파싱 매핑
어드민 로그보드 로딩 시 로그 한 줄을 각각의 객체 필드로 파싱하기 위해 본사는 아래 정규식을 사용해 역직렬화합니다.

```javascript
const logRegex = /^\[([^\]]+)\]\s+\[([^\]]+)\]\s+\[IP:\s+([^\]]+)\]\s+\[APPROVED:\s+([^\]]+)\]\s+-\s+(.*)$/;
```

*   **Group 1**: 연동 시각 (`2026-06-11 07:58:12`)
*   **Group 2**: 매장 코드 (`STORE-1002`)
*   **Group 3**: IP 정보 (`192.168.0.24`)
*   **Group 4**: 승인 판정 (`APPROVED: PASS` 또는 `APPROVED: FAIL`)
*   **Group 5**: 상세 메시지 (`정식 승인 통과 (마포점 / BASIC)`)

## 📌 4. 로그 등급별 판정 및 대응 행동 규칙

| 등급 | 판정 상태 및 문구 | 원인 및 시스템 영향 | 대응 조치 |
| :--- | :--- | :--- | :--- |
| 🟢 **정상**<br>`(PASS)` | `APPROVED: PASS` | • 커피머신 로컬서버 가동 및 인증 완료 상태 | • 별도 대응 조치 없음 (모니터링 유지) |
| 🟡 **주의**<br>`(WARN)` | `APPROVED: FAIL` (EXPIRED) | • 라이선스 기간 만료 상태이나 7일간의 오프라인 임시 유예 토큰으로 동작 중 | • 해당 점주에게 알림톡으로 기간 연장 및 결제 안내 전송 |
| 🔴 **위험**<br>`(DANGER)` | `APPROVED: FAIL` (SUSPENDED)<br>또는 `INVALID_KEY` | • 미인증 접근 시도, 강제 차단 계정 접속, 또는 디도스(DDoS) 의심 해킹 트래픽 발생 | • DDoS 차단 방어벽 IP 대역 차단 실행<br>• DB 커넥션 고갈 위협 여부 점검 및 즉각적인 관리자 수동 검증 기동 |
