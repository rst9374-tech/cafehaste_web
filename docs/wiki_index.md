## 📝 2. 카파시 기스트 (Karpathy Gist) 기반 에이전트 지침
안드레 카파시(Andrej Karpathy)가 제시한 LLM 코딩 및 에이전트 지식 주입 가이드를 기반으로 설계되었습니다.
1. **컨텍스트 밀도 극대화**: LLM은 수많은 파편화된 파일을 조각조각 읽는 것보다, 연관된 도메인 규칙이 하나의 구조화된 문서에 압축되어 제공될 때 가장 코딩 오작동율이 적습니다.
2. **500줄 제한 규정과의 시너지**: 소스 코드의 줄 수를 500줄 이하로 철저히 격리하고, 규칙은 위키(Wiki)로 통합 관리하여 LLM이 코드를 읽고 분석할 때 발생하는 비용과 망각(Forgetfulness)을 원천 방어합니다.

## ✍️ 3. 마크다운(MD) 작성 및 인덱싱 규칙
옵시디언과 에이전트가 본 문서를 가장 정확하게 해석하도록 돕는 마크다운 스타일 규약입니다.
*   **양방향 링크 (`[[파일명]]`)**: 문서 간의 관계를 수동으로 연결해 주기 위해, 연관 문서 언급 시 반드시 대괄호 이중 괄호(`[[파일명]]`)를 사용하여 옵시디언 그래프 신경망에 반영되게 합니다.
*   **제목 계층 준수**: `#` (문서 제목), `##` (대단원 섹션), `###` (소분류) 규칙을 지켜 구조화합니다.
*   **코드 블록 명시**: 예시 코드 삽입 시 반드시 펜싱 코드 블록(예: ` ```tsx ` or ` ```json `)을 써서 언어 구분을 명확히 지원합니다.
